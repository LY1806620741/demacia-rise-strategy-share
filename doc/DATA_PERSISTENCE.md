# 数据持久化功能说明\n\n## 🔧 已修复的问题\n\n用户关闭浏览器后，数据会丢失的问题已解决。现在实现了完整的数据持久化机制。\n\n## 📦 什么被持久化了？\n\n### 1. 策略数据 ✅\n- **策略标题、描述、目标、阵容、科技等信息**\n- 存储位置: IndexedDB `strategies` 表\n- 持久化时机: 创建策略时立即保存\n\n### 2. 投票数据 ✅\n- **每个策略的赞数和踩数**\n- **投票者追踪信息（用于防止重复计数）**\n- 存储位置: IndexedDB `vote_history` 表\n- 持久化时机: 用户投票时立即保存\n\n### 3. 官方推荐 ✅\n- **官方初始化的 4 个阵容方案**\n- 存储位置: IndexedDB `lineups` 表\n- 持久化时机: 首次启动时初始化\n\n## 🔄 工作流程\n\n### 启动时 (页面加载/浏览器打开)\n```\nstart()\n  ↓\n初始化 IndexedDB\n  ↓\n从 IndexedDB 加载所有策略\n  ↓\n将策略恢复到 WASM 内存\n  ↓\n从 IndexedDB 加载投票历史\n  ↓\n恢复 localVotes Map\n  ↓\n显示完整的数据 + 投票计数 ✅\n```\n\n### 创建策略时\n```\n用户填表单并点击 \"发布策略\"\n  ↓\nsubmitStrategy()\n  ↓\ncreate_strategy() [WASM: 添加到内存索引]\n  ↓\nsaveStrategyToDB() [IndexedDB: 持久化]\n  ↓\nsaveLineupToDB() [IndexedDB: 保存阵容方案]\n  ↓\n广播到 P2P 网络\n  ↓\nUI 刷新显示 ✅\n```\n\n### 投票时\n```\n用户点击投票按钮\n  ↓\nvoteOnStrategy(strategyId, isLike)\n  ↓\nbroadcastVote() [发送到 P2P 网络]\n  ↓\nhandleVoteEvent() [更新内存计数]\n  ↓\nupdateVoteInDB() [保存投票到 IndexedDB]\n  ├─ 更新策略的 likes/dislikes 数字\n  └─ 保存投票详情到 vote_history\n  ↓\nsaveStrategyToDB() [再次持久化计数]\n  ↓\nrenderStrategies() [UI 刷新显示 ✅]\n```\n\n## 📊 IndexedDB 架构\n\n### 数据库: DemaciaRise\n\n#### 表 1: strategies\n```javascript\n{\n  keyPath: 'id',\n  数据:\n  {\n    id: \"uuid\",\n    title: \"策略名称\",\n    description: \"描述\",\n    target_hero: \"敌人\",\n    counter_lineup: \"阵容\",\n    counter_tech: \"科技\",\n    likes: 数字,        // 投票计数\n    dislikes: 数字,     // 投票计数\n    score: 数字,        // 评分\n    created: 时间戳\n  }\n}\n```\n\n#### 表 2: lineups\n```javascript\n{\n  keyPath: 'id',\n  索引: 'target' (目标敌人)\n  数据:\n  {\n    id: \"uuid\",\n    name: \"方案名称\",\n    target: \"目标敌人\",\n    units: [\"unit1\", \"unit2\"],   // 兵种 ID 数组\n    description: \"描述\",\n    likes: 数字,\n    dislikes: 数字,\n    created: 时间戳\n  }\n}\n```\n\n#### 表 3: vote_history (新增) ⭐\n```javascript\n{\n  keyPath: 'strategyId',\n  数据:\n  {\n    strategyId: \"uuid\",\n    likes: [\n      [\"nodeId-A\", timestamp],\n      [\"nodeId-B\", timestamp]\n    ],\n    dislikes: [\n      [\"nodeId-C\", timestamp]\n    ],\n    timestamp: 最后更新时间\n  }\n}\n```\n\n## 🧪 测试持久化功能\n\n### 测试 1: 策略持久化\n\n```bash\n# 步骤 1: 打开浏览器\n# console.log('✅ IndexedDB 可用');\n  console.log('存储:', Array.from(req.result.objectStoreNames));\n};\nreq.onerror = () => {\n  console.error('❌ IndexedDB 不可用:', req.error);\n};\n```\n\n### 问题: 投票计数不一致\n\n**原因**: localVotes 和 IndexedDB 不同步\n\n**解决方案**:\n```javascript\n// 手动重新加载投票数据\nawait loadVoteHistoryFromDB();\nconsole.log('✅ 投票数据已重新加载:', localVotes);\n```\n\n### 问题: IndexedDB 占用空间过大\n\n**原因**: 长期使用积累了过多历史数据\n\n**解决方案**:\n```javascript\n// 清除 IndexedDB (谨慎操作)\nconst deleteDB = indexedDB.deleteDatabase('DemaciaRise');\ndeleteDB.onsuccess = () => console.log('✅ IndexedDB 已清除');\n\n// 然后刷新页面\nlocation.reload();\n```\n\n### 问题: 投票在某个标签页消失\n\n**原因**: 该标签页的 IndexedDB 事务未完成\n\n**解决方案**:\n- 检查浏览器控制台是否有错误\n- 重启该标签页\n- 手动调用 `broadcastVote()` 重新广播\n\n## 💾 数据恢复指令\n\n### 手动恢复所有数据\n\n```javascript\n// 在浏览器控制台执行\n\n// 1. 重新初始化数据库\nasync function restore() {\n  console.log('开始恢复...');\n  \n  // 清空 WASM 策略索引\n  // (需要 WASM API 支持，暂无直接方法)\n  \n  // 2. 从 IndexedDB 重新加载\n  const strategies = await loadStrategiesFromDB();\n  for (const strat of strategies) {\n    create_strategy(\n      strat.id,\n      strat.title,\n      strat.description,\n      strat.target_hero,\n      strat.counter_lineup,\n      strat.counter_tech\n    );\n  }\n  \n  // 3. 恢复投票\n  await loadVoteHistoryFromDB();\n  \n  // 4. 刷新 UI\n  renderStrategies();\n  \n  console.log(`✅ 恢复完成: ${strategies.length} 个策略`);\n}\n\nrestore();\n```\n\n### 导出数据为 JSON\n\n```javascript\n// 备份所有数据\nasync function exportData() {\n  const strategies = await loadStrategiesFromDB();\n  const lineups = await queryDB('lineups', 'all');\n  \n  const backup = {\n    timestamp: new Date().toISOString(),\n    strategies,\n    lineups\n  };\n  \n  // 下载 JSON 文件\n  const blob = new Blob([JSON.stringify(backup, null, 2)], \n    { type: 'application/json' });\n  const url = URL.createObjectURL(blob);\n  const a = document.createElement('a');\n  a.href = url;\n  a.download = `demacia-backup-${Date.now()}.json`;\n  a.click();\n  \n  console.log('✅ 数据已导出');\n}\n\nexportData();\n```\n\n## 📈 性能指标\n\n| 操作 | 耗时 | 备注 |\n|------|------|------|\n| 保存 1 个策略 | ~5ms | IndexedDB 写入
