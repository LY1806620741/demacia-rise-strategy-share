创建与共享策略\n\n```\n步骤 1: 打开 3 个浏览器标签页\n  - 标签页 A, B, C 都访问 http://localhost:8000\n\n步骤 2: 在标签页 A 中创建策略\n  - 标题: \"对抗达瑞斯的坦克防线\"\n  - 敌人: 达瑞斯\n  - 阵容: 选择 \"波比\", \"盖伦\", \"奎因\"\n  - 科技: 选择 \"盾牌强化\", \"反伤技能\"\n  - 点击 \"发布策略\"\n  \n结果:\n  ✅ 阵容列表中出现新策略\n  ✅ 标签页 B 和 C 立即同步显示\n```\n\n### 🔍 场景 2: 跨节点搜索\n\n```\n步骤 1: 标签页 A 中创建 2-3 个本地策略\n\n步骤 2: 标签页 B 中搜索 \"达瑞斯\"\n  - 结果中同时出现:\n    - 标签页 A 的策略 (标注: \"[本地)\" 或 \"[peer:xxxxx]\")\n    - 官方推荐数据\n    \n步骤 3: 注意到结果来源标签\n  - [local]: 本地节点的策略\n  - [peer:xxxxxxxx]: 对等节点的策略 (8 位节点 ID)\n  - [strategy]: 官方推荐方案\n```\n\n### 👍 场景 3: 投票权重提升排名\n\n```\n步骤 1: 标签页 A 创建策略 X (相关性分: 10)\n步骤 2: 标签页 B 创建策略 Y (相关性分: 15)\n\n初始搜索结果:\n  #1. 策略 Y (分数 15)\n  #2. 策略 X (分数 10)\n\n步骤 3: 在标签页 C 中搜索相同关键词\n  - 对策略 X 点赞 10 次 ✓✓✓...\n  - 对策略 Y 点赞 1 次 ✓\n  \n新的排序:\n  weighted_X = 10 + (10 * 0.5) * 100 = 510 ✨ 排名 #1\n  weighted_Y = 15 + (1 * 0.5) * 100 = 65  排名 #2\n  \n验证: 社群投票超越相关性!\n```\n\n### ❌ 场景 4: 投票反转\n\n```\n步骤 1: 用户 A 对策略 X 点赞\n  - 赞数: 1\n  - localVotes[X].likes.set(nodeId_A, timestamp)\n\n步骤 2: 用户 A 改变主意，点踩\n  - localVotes[X].likes.delete(nodeId_A)\n  - localVotes[X].dislikes.set(nodeId_A, new_timestamp)\n  - 赞数: 0, 踩数: 1 ✓\n  \n结果: 投票计数正确反转\n```\n\n### 📊 场景 5: 索引聚合与 TTL\n\n```\n步骤 1: 打开 2 个标签页\n  - 标签页 A: 创建 5 个策略\n  - 标签页 B: 等待接收\n\n观察 (在开发者控制台):\n  时刻 T0: peerIndices 为空\n  时刻 T5: 第一次广播 → peerIndices[nodeId_A] 存储 5 个策略 ✓\n  时刻 T10: 第二次广播 → 更新 timestamp\n  \n步骤 2: 关闭标签页 A (模拟节点离线)\n\n观察:\n  时刻 T15: TTL 检查 (15000 - 10000 = 5000ms 超时)\n  → peerIndices[nodeId_A] 自动清理\n  → 后续搜索不再包含该节点的策略 ✓\n```\n\n## UI 交互指南\n\n### 常见问题排查\n\n### Q: 搜索结果中看不到其他标签页的策略\nA: \n1. 检查 P2P 状态指示板: 节点数是否 > 1?\n2. 打开控制台: `console.log(peerIndices)` 检查缓存\n3. 手动触发: `broadcastIndexShare()` 再搜索\n4. 等待 5-10 秒 (索引广播周期)\n\n### Q: 投票数据在刷新后丢失\nA:\n1. 检查 IndexedDB: \n   ```javascript\n   const strats = await loadStrategiesFromDB();\n   console.log('IndexedDB 中的策略数:', strats.length);\n   ```\n2. 确保启用了 IndexedDB (`initIndexedDB()` 在启动时调用)\n3. 打开浏览器 DevTools → Application → IndexedDB → DemaciaRise\n\n### Q: 投票按钮无反应\nA:\n1. 检查 JavaScript 语法: `node -c app.js`\n2. 在控制台测试: `voteOnStrategy('test-id', true)`\n3. 查看浏览器控制台是否有错误\n4. 清除浏览器缓存 (Cmd+Shift+R)\n\n### Q: 多个标签页之间不同步\nA:\n1. 检查 storage 事件监听器是否正常:\n   ```javascript\n   window.addEventListener('storage', (e) => {\n     console.log('Storage 事件:', e.key, e.newValue.slice(0, 50));\n
