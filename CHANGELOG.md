# 变更日志 (Changelog)\n\n## [1.0.0] - 2024-01-22: 去中心化 P2P 搜索发布\n\n### 🎉 重大特性\n\n#### 新增: 投票同步机制\n- **功能**: 用户投票 (点赞/点踩) 实时广播到所有对等节点\n- **实现**:\n  - `broadcastVote(strategyId, isLike)` - 投票事件广播\n  - `handleVoteEvent(msg)` - 接收和处理投票\n  - `updateVoteInDB(strategyId, votes)` - 投票持久化\n- **文件**: `app.js` (新增 ~80 行)\n- **影响**: 支持跨网络共识权重计算\n\n#### 新增: 索引分享与聚合\n- **功能**: 周期性广播本地策略索引，聚合对等节点的策略用于搜索\n- **实现**:\n  - `broadcastIndexShare()` - 每 5 秒广播本地索引\n  - `handleIndexShare(msg)` - 缓存对等节点索引\n  - `peerIndices` Map - 对等节点策略存储\n- **文件**: `app.js` (新增 ~60 行)\n- **影响**: 启用跨节点统一搜索\n\n#### 新增: 聚合搜索引擎\n- **功能**: 将本地搜索结果与对等节点结果合并，基于投票权重排序\n- **实现**:\n  - `aggregateSearch(q, limit)` - 交叉搜索与聚合\n  - `calculateRelevance(q, strategy)` - 对等节点 BM25 计算\n  - 加权公式: `weighted_score = rank + (votes.likes * 0.5 - votes.dislikes * 0.3) * 100`\n- **文件**: `app.js` (新增 ~100 行)\n- **影响**: 搜索结果兼顾相关性和社群共识\n\n#### 新增: 投票计数显示\n- **功能**: UI 中显示投票按钮与实时计数\n- **实现**:\n  - `renderSearchHitsWithVotes(hits, q)` - 搜索结果渲染 (带投票)\n  - `renderStrategies()` 改进 - 显示投票数\n  - `window.voteOnStrategy()` - UI 事件处理入口\n- **文件**: `app.js` (修改 ~50 行) + `app.css` (新增 ~40 行)\n- **影响**: 用户可直接交互投票按钮\n\n### 📝 文件修改详情\n\n#### app.js
