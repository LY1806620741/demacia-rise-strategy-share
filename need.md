# 项目需求文档（Need.md）
# 英雄联盟：德玛西亚的崛起 - 策略模拟器
## 核心定位
一个**无服务器、纯浏览器 P2P 分布式、社区驱动**的英雄联盟战术策略模拟器。
**官方定基准，社区定策略；数据永不丢失，智慧共同进化。**

---

# 1. 功能需求
## 1.1 数据架构
- **官方基准数据**
  - 英雄、建筑、资源、技能等基础数据
  - 只读、GitHub 托管、可信源
- **野生策略数据**
  - 用户提交的战术、阵容、对抗方案、运营路线
  - 分布式存储、P2P 同步、社区共识驱动

## 1.2 核心系统
- **资源模拟系统**
  - 金币、人口、光盾素材、军备值
  - 建筑与科技树、用户可扩展路线
- **战斗模拟系统**
  - 阵容构建、战术指令、PVE / 异步 PVP 推演
  - 野生对策库：针对强势英雄的最小成本解法
- **共识与推荐系统**
  - 点赞/点踩形成策略权重
  - 高权重策略自动提升曝光
  - 官方数据 > 重要对策 > 高赞 > 普通

## 1.3 P2P 网络需求
- **浏览器原生 P2P（WebRTC）**
- **节点无需拉取全量数据**
- **检索式查询**（关键词、英雄、战术类型）
- **分布式存储 + 分布式检索**
- **高权重/重要数据自动扩散、多节点缓存**
- **低价值数据不占本地存储空间**

## 1.4 安全与防攻击需求
- **无工作量证明（PoW）、不浪费性能**
- 防刷赞、防机器人、防女巫攻击
- 防重复、防垃圾内容、防篡改
- 采用：冷却机制、唯一身份、内容哈希、信任加权、链式哈希

## 1.5 权重与推荐需求
- 权重公式：
  ```
  得分 = 官方标识*1000 + 点赞*2 - 点踩*3 + 信任值*5 + 新鲜度*10 + 重要标记*50
  ```
- **重要数据自动提升权重**（版本答案、关键对抗策略）
- 搜索结果按权重排序
- 高权重数据自动缓存更多节点

---

# 2. 技术架构需求
## 2.1 技术栈
- **前端**：原生 HTML / CSS / JS
- **核心逻辑**：Rust + WASM
- **P2P**：WebRTC 浏览器点对点
- **存储**：本地 IndexedDB + P2P 分布式缓存
- **同步**：Gossip 协议风格广播

## 2.2 架构特性
- 无中心化服务器
- 无数据库
- 节点自动发现、自动组网
- 数据高可用、防单点失效
- 轻量级节点，手机/浏览器均可运行

## 2.3 网络特性
- 检索优先，非同步全量数据
- 高权重数据自动冗余
- 查询全网广播，节点按需返回
- 跨标签页、跨设备、局域网自动同步

---

# 3. 非功能需求
- **高性能**：WASM 计算，前端不卡顿
- **安全**：防攻击、防刷赞、防篡改
- **绿色**：无挖矿、无功耗浪费
- **去中心化**：永不宕机、无运营成本
- **可扩展**：支持后续 P2P 共识、信誉系统、PWA 离线

- 策略信誉与权重进化
